# Content and Cartridge Rights Notice

PixelForge Studio is intended to be an open-source game creation forge.

The source code for the engine, tooling, validators, templates, and developer utilities is licensed under the MIT License unless a file clearly says otherwise.

Game content is different.

Game cartridges, sample adventures, art, writing, music, characters, logos, story worlds, brand names, and 369 PocketGames release assets may have their own rights boundaries. Do not assume that every asset or story element in this repository is free to reuse in commercial projects.

## Practical rule

- You may use the PixelForge Studio source code under the MIT License.
- You may study sample cartridges to learn the format.
- You may fork templates to build your own original cartridges.
- You should not copy protected characters, story text, game worlds, logos, music, art, or 369 PocketGames branding into your own commercial release unless a cartridge or asset file explicitly grants that permission.

## Community cartridge submissions

By submitting a community cartridge, you should only include material that you created, have permission to use, or can clearly license for the project.

A submission should include a manifest that states:

- who made the cartridge
- what license applies to the cartridge content
- whether third-party assets are included
- whether the cartridge can be remixed
- whether the cartridge can be considered for a future 369 PocketGames release

## 369 PocketGames

369 PocketGames is the polished mobile/premium label for small, complete games with no ads, no subscriptions, no loot boxes, no energy timers, and no predatory in-app purchases.

The 369 PocketGames name, release packaging, premium store assets, and final commercial cartridge builds may be protected separately from the open-source PixelForge Studio code.

## Default stance

When in doubt:

- code is open under MIT
- templates are for learning and building original work
- story/art/music/brand assets need explicit permission
- contributors should keep receipts for asset ownership and permissions
